# Беклог

Цей документ зберігає пріоритизовану роботу, яка ще не стала активною.

## Модель статусів

Для кожного елемента використовуй один із таких статусів:

- `Draft`
- `Ready`
- `Blocked`
- `Done`
- `Archived`

## Шаблон елемента

Використовуй компактний формат для кожного елемента беклогу:

```md
- [Status] Коротка назва
  - Чому це важливо:
  - Ознака прийняття:
  - Посилання:
```

## Правило супроводу

- Тримай список пріоритизованим зверху вниз.
- Переміщуй активні плани виконання в [`active/`](active/README.md).
- Переміщуй завершену або замінену роботу в [`archive/`](archive/README.md).
