# Архів планування

У цьому каталозі зберігай завершені, скасовані або замінені документи планування.

## Правило архівування

- Зберігай оригінальний вміст плану.
- Додавай коротку примітку на початку, якщо план було скасовано або замінено.
- Зберігай стабільними назви файлів верхнього рівня під час переміщення їх з [`active/`](../active/README.md).
- Зберігай стабільними назви канонічних директорій епіків і сторі під час переміщення дерев планування з `active/` до `archive/`.
- Зберігай стабільними канонічні внутрішні назви файлів в архівованих деревах планування: `{id}_{short_title}.md`, необов'язковий `breakdown.md` і файли завдань `{order}_{id}_{short_title}.md`.
- Переміщення до архіву не повинні змінювати значення `Planning ID` або локальний порядок сусідніх елементів, якщо примітка архіву явно не документує навмисний знімок перепріоритизації.
