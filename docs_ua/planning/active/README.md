# Активні плани

Цей каталог зберігає активні елементи планування.

## Структура

- Плани постачання або дослідження верхнього рівня, які охоплюють кілька віх, залишаються в цьому каталозі як файли з префіксом дати.
- Кожен активний епік живе у власній директорії з назвою `{order}_{id}_{short_title}`.
- Кожна директорія епіка містить один канонічний файл епіка з назвою `{id}_{short_title}.md`.
- Кожна сторі живе безпосередньо у директорії свого батьківського епіка як `{order}_{id}_{short_title}`.
- Кожна директорія сторі містить один канонічний файл сторі з назвою `{id}_{short_title}.md`.
- Кожне завдання живе як Markdown-файл у батьківській директорії сторі `tasks/` з назвою `{order}_{id}_{short_title}.md`.

Приклад:

```text
docs_ua/planning/active/
  2026-03-14-mvp-delivery-plan.md
  01_0001_foundations_ready/
    0001_foundations_ready.md
    01_0002_runtime_package_contract/
      0002_runtime_package_contract.md
    03_0012_storage_foundation/
      0012_storage_foundation.md
      tasks/
        01_0013_storage_surface_alembic.md
```

## Правила найменування

- Використовуй lowercase ASCII `snake_case` для всіх slug у назвах директорій і файлів.
- Використовуй стабільні двозначні числа `order` для пріоритету виконання в межах одного набору сусідів: директорії епіків упорядковуються в `active/`, директорії сторі впорядковуються в межах свого батьківського епіка, а файли завдань упорядковуються в межах своєї батьківської сторі.
- Використовуй стабільні чотиризначні значення `Planning ID` для канонічної ідентичності епіків, сторі й завдань у `active/` і `archive/`.
- Директорії епіків і сторі використовують формат `{order}_{id}_{short_title}`.
- Файли епіків і сторі використовують формат `{id}_{short_title}.md`.
- Файли завдань використовують формат `{order}_{id}_{short_title}.md`.
- `Planning ID` ніколи не повинен змінюватися після призначення.
- Перепріоритизація може змінювати локальний `order`, але не повинна змінювати `Planning ID`.
- Не перенумеровуй наявні елементи лише для того, щоб закрити прогалини після видалення або переміщення в архів; перенумеровуй лише тоді, коли свідомо змінюєш пріоритет виконання.

## Канонічні файли та ID

- Директорії епіків повинні містити рівно один канонічний документ епіка з назвою `{id}_{short_title}.md`.
- Директорії сторі повинні містити рівно один канонічний документ сторі з назвою `{id}_{short_title}.md`.
- Файли завдань є канонічними безпосередньо й не мають окремої директорії завдання на кожний елемент.
- Записуй `Planning ID` безпосередньо під заголовком документа в кожному канонічному документі епіка, сторі й завдання.
- Записуй `Milestone` безпосередньо під заголовком документа лише для документів епіків.
- `breakdown.md` є необов'язковим для директорій епіків і сторі та повинен існувати лише тоді, коли потрібна додаткова деталізація дослідження або декомпозиції.
- Створюй `tasks/` лише тоді, коли сторі декомпозовано на виконувані завдання.
- Не створюй піддиректорії для завдань.
- Файли планів верхнього рівня пов'язуються через шлях і не отримують значень `Planning ID`.
- Призначай наступний `Planning ID` як `max(наявні active/archive Planning ID) + 1`.
