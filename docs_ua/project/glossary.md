# Глосарій

Використовуй цей документ для визначення специфічних для проєкту термінів, скорочень і доменної мови.

## Терміни

- `Telegram-ggregator`: робоча назва репозиторію та сервісу для репостингу з Telegram.
- `Source channel`: Telegram-канал або чат, з якого сервіс читає вхідні повідомлення.
- `Target channel`: Telegram-канал, що отримує репостований вміст.
- `Filter rule`: налаштоване правило include або exclude на основі ключових слів чи regex-патернів.
- `Typed include rule`: правило include, яке містить `pattern`, `event_type` і `event_signal`.
- `Candidate message`: вихідне повідомлення, що збіглося з одним типізованим правилом include і очікує агрегації на рівні події.
- `Event signal`: семантика життєвого циклу, прив'язана до типізованого правила include. MVP підтримує `start` і `clear`.
- `Event type`: налаштована логічна категорія, наприклад `ballistic`, `missile` або `drone`.
- `Logical event`: згрупована подія загрози, яка може містити кілька кандидатів із різних джерел.
- `Canonical source`: перший кандидат, який відкриває логічну подію й стає джерелом публікації для цієї події.
- `Orphan clear`: кандидат `clear`, який не відповідає жодній поточній відкритій події того самого типу.
- `Repost`: публікація вибраного вихідного вмісту в цільовий канал.
- `Attribution`: посилання на джерело, яке додається до репосту, зазвичай із назвою джерела та лінком на пост.
- `Deduplication`: логіка, що запобігає повторній публікації того самого вихідного повідомлення або тієї самої логічної події.
- `FloodWait`: період очікування, який Telegram накладає після вичерпання ліміту.
- `User session`: збережена автентифікована сесія Telegram-акаунта, що використовується для доступу через MTProto.
- `Dry run`: режим виконання, у якому збіги логуються, але не публікуються.

Додавай нові записи, коли термін можуть по-різному трактувати інженери, оператори або стейкхолдери.
