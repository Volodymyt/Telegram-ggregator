# Рекомендації для ADR

Записи архітектурних рішень фіксують рішення, які мають залишатися доступними з часом.

## Коли створювати ADR

Створи ADR, коли рішення:

- суттєво змінює архітектуру,
- вводить нову залежність або інтеграцію,
- встановлює довготривале процесне чи операційне правило,
- закриває компроміс, до якого інакше доведеться повертатися знову й знову.

## Угода про найменування

Використовуй послідовні файли з короткою назвою:

- `0001-record-name.md`
- `0002-record-name.md`

## Рекомендований шаблон

```md
# ADR 0001: Title

## Status

Proposed | Accepted | Superseded

## Context

## Decision

## Consequences
```
