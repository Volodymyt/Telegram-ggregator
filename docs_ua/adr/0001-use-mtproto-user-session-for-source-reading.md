# ADR 0001: Використовувати користувацьку MTProto-сесію для читання джерел

## Статус

Прийнято

## Контекст

Сервіс має читати повідомлення з налаштованого списку Telegram-каналів і чатів-джерел.

Telegram Bot API недостатньо для цієї вимоги, коли канали-джерела є зовнішніми, а бота не додано явно з потрібними дозволами. Бот не може поводитися як RSS-рідер, що підписується на довільні Telegram-канали.

Тому для MVP потрібен підхід із Telegram-клієнтом, який може читати канали, доступні автентифікованому користувацькому акаунту.

## Рішення

Сервіс використовуватиме Telethon зі збереженою Telegram-користувацькою сесією для читання повідомлень із джерел через MTProto.

## Наслідки

- Середовищу виконання потрібні `TG_API_ID` і `TG_API_HASH`.
- Перед звичайною роботою демона потрібен одноразовий крок авторизації акаунта.
- Файл сесії треба зберігати поза образом, зазвичай у змонтованому томі даних.
- Секрети треба передавати через змінні середовища, а не вшивати в контейнерний образ.
- Операційна документація має покривати перший вхід і збереження сесії.
