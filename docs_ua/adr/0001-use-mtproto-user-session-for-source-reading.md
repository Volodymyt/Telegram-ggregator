# ADR 0001: Використовувати користувацьку сесію MTProto для читання джерел

## Статус

Прийнято

## Контекст

Сервіс має читати повідомлення з налаштованого списку вихідних Telegram-каналів і чатів.

Telegram Bot API недостатньо для цієї вимоги, коли вихідні канали є зовнішніми, а бота явно не додано з потрібними правами. Бот не може поводитися як RSS-рідер, що підписується на довільні Telegram-канали.

Тому для MVP потрібен підхід із Telegram-клієнтом, який може читати канали, доступні автентифікованому користувацькому акаунту.

## Рішення

Сервіс використовуватиме Telethon із збереженою Telegram-користувацькою сесією для читання вихідних повідомлень через MTProto.

## Наслідки

- Середовищу виконання потрібні `TG_API_ID` і `TG_API_HASH`.
- Перед звичайною роботою демона потрібен одноразовий крок авторизації акаунта.
- Файл сесії треба зберігати поза образом, зазвичай у змонтованому томі даних.
- Секрети треба передавати через змінні середовища, а не вшивати в контейнерний образ.
- Операційна документація має покривати перший вхід і збереження сесії.
