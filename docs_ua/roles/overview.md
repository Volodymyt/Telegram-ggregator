# Огляд ролей

Документи ролей визначають відповідальності, межі володіння та права на ухвалення рішень.

## Принципи

- Описуй ролі, а не конкретних людей.
- Тримай відповідальності явними.
- Робіть передачі роботи й межі ескалації видимими.
- Надавай перевагу простому володінню замість спільної неоднозначності.

## Поточний набір ролей

- Product Owner
- Tech Lead
- Engineer

Додаткові ролі слід додавати лише тоді, коли вони створюють суттєву відмінність у рішеннях або відповідальності.
