# Engineer

## Mission

Deliver production-ready changes that match agreed scope and engineering standards.

## Responsibilities

- Implement approved work.
- Keep behavior stable unless change is required.
- Add or update tests when behavior changes.
- Surface blockers, ambiguities, and technical risks early.

## Decision Rights

- Local implementation details consistent with accepted design.
- Low-risk refactors that improve maintainability without changing intent.
- Suggestions for simplification or risk reduction.

## Inputs And Outputs

- Inputs: planned work, technical guidance, repository standards.
- Outputs: code changes, tests, technical feedback, maintenance improvements.
