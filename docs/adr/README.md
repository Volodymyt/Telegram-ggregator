# ADR Guidelines

Architecture Decision Records capture decisions that should remain discoverable over time.

## When To Create An ADR

Create an ADR when a decision:

- changes architecture in a meaningful way,
- introduces a new dependency or integration,
- sets a lasting process or operational rule,
- resolves a tradeoff that might otherwise be revisited repeatedly.

## Naming Convention

Use sequential files with a short title:

- `0001-record-name.md`
- `0002-record-name.md`

## Suggested Template

```md
# ADR 0001: Title

## Status

Proposed | Accepted | Superseded

## Context

## Decision

## Consequences
```
