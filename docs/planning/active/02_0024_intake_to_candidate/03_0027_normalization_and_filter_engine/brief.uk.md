# Бриф

## Коротко

Потрібно винести спільну нормалізацію тексту для класифікації та зафіксувати правила фільтрації для повідомлень і медіа-підписів.
Також потрібно додати обов’язкове runtime-поле для часу, після якого класифікація вважається застарілою.

## Що потрібно зробити

- [ ] Додати спільні хелпери нормалізації з `NFKC`, нижнім регістром, обрізанням пробілів, згортанням повторних пробілів і заміною `ё` на `е`.
- [ ] Застосувати include та exclude-правила до тексту повідомлення і до медіа-підписів через спільну нормалізацію.
- [ ] Реалізувати обхід груп у порядку конфігурації з підтримкою `any`, `all` і правила `first matching group wins`.
- [ ] Розширити YAML runtime-контракт полем `classification_stale_after_seconds` і зробити його обов’язковим додатним цілим числом.
- [ ] Оновити типізовану модель, семантичну валідацію та канонічні приклади конфігу для нового runtime-поля.
- [ ] Додати перевірки для toggles нормалізації, порядку груп, режиму `all`, порожнього `exclude` і валідації `classification_stale_after_seconds`.
