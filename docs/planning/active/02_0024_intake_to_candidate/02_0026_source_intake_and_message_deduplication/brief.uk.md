# Бриф

## Коротко

Потрібно провести конфігуровані Telegram-джерела через канонічний runtime шлях і зберігати кожне нове повідомлення рівно один раз. У чергу обробки мають потрапляти лише вперше побачені рядки.

## Що потрібно зробити

- [ ] Підключити вибрані Telegram-джерела через канонічний runtime та reader boundary.
- [ ] Замапити вхідні події Telethon в єдиний intake-формат із `raw_text`, ознаками медіа, часовою міткою та source-метаданими.
- [ ] Зберігати нові `tg_message` з початковими станами `classification_status='pending'`, `aggregation_status='new'` і `publish_status='new'`.
- [ ] Гарантувати дедуплікацію за `(source_chat_id, source_message_id)` без повторного enqueue для дубльованих доставок.
- [ ] Enqueue-ити лише щойно створені рядки та залишати повторні доставки безпечним no-op.
- [ ] Зберегти в intake-рядах текстову поверхню повідомлення в `raw_text`, використовуючи caption для медіа-повідомлень.
