# Backlog

This document stores prioritized work that is not yet active.

## Status Model

Use one of the following statuses for each item:

- `Draft`
- `Ready`
- `Blocked`
- `Done`
- `Archived`

## Item Template

Use a compact format for each backlog item:

```md
- [Status] Short title
  - Why it matters:
  - Acceptance signal:
  - Links:
```

## Maintenance Rule

- Keep the list prioritized from top to bottom.
- Move active execution plans into `active/`.
- Move completed or superseded work into `archive/`.
